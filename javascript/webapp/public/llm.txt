# simplehook

> Webhooks that just work. One line of code.

simplehook gives developers and AI agents stable, persistent webhook URLs that forward to localhost — no CLI tools, no tunnels, no URL that changes every session.

## What it does

simplehook replaces tools like ngrok for webhook development. You add one line of code to your app (Express, Fastify, Flask, FastAPI, Hono), and webhooks from providers like Stripe, GitHub, Twilio, and Shopify are delivered to your local development server over WebSocket. Webhook signature headers are preserved — signature verification works as normal.

## Key features

- Stable webhook URLs that never change
- WebSocket-based delivery (outbound only, no port forwarding)
- One-line SDK integration for Express, Fastify, Hono (Node.js), Flask, FastAPI (Python)
- AI agent HTTP pull API (instant, long-poll, or SSE stream)
- CLI for route and listener management
- Automatic retries with exponential backoff
- Request/response passthrough (caller gets your actual response)
- Targeted routing: assign webhook paths to specific named connections
- Connections auto-register when SDK connects with a name

## Two consumption modes

### Developer mode (SDK)
Add one line to your Express/Fastify/Flask/FastAPI app. Webhooks arrive as normal HTTP requests to your local routes. Your response flows back to the original caller.

### Agent mode (CLI / HTTP API)
AI agents pull webhooks via simple HTTP — instant poll, long-poll, or SSE stream. No SDK needed, just HTTP requests with an API key. Or use the CLI: `npx @simplehook/cli pull --wait --path "/stripe/*"`

## Two delivery modes

### Queue mode (default)
Returns 200 to the provider immediately. Events queue and forward to your app with automatic retries. Best for Stripe, Shopify, most providers.

### Passthrough mode
Proxies the request to your app and returns your actual response to the provider. Best for Twilio (TwiML), GitHub status checks.

## Connections (listeners)

Run multiple SDK instances and route events to specific ones. Pass a name when connecting:
`listenToWebhooks(app, process.env.SIMPLEHOOK_KEY, "dev")`

The connection auto-registers on first connect. Assign routes to connections via dashboard or CLI:
`npx @simplehook/cli routes create /stripe --listener dev`

## SDKs

- Node.js: `npm install @simplehook/express` (Express), `npm install @simplehook/fastify` (Fastify), `npm install @simplehook/hono` (Hono)
- Python: `pip install simplehook-flask` (Flask), `pip install simplehook-fastapi` (FastAPI)
- CLI: `npm install -g @simplehook/cli`
- AI Agent SDK: `npm install @simplehook/core` (SimplehookAgent class)
- Mastra: `npm install @simplehook/mastra`

## CLI commands

- `simplehook pull` — Pull webhook events (instant, --wait, --stream)
- `simplehook status` — Queue health, cursors, connected listeners
- `simplehook routes` — List routes
- `simplehook routes create <path> --listener <id>` — Create targeted route
- `simplehook listeners` — List connections with status

## API reference

### Agent pull API

GET https://hook.simplehook.dev/api/agent/pull
Authorization: Bearer ak_...

Query parameters:
- wait=true — long-poll (blocks until a webhook arrives, up to 25s)
- stream=true — SSE stream of incoming webhooks
- path=/stripe/* — filter by webhook path (glob pattern)
- listener_id=my-agent — consumer identity (each has own cursor)
- n=10 — number of events to return (1-100)

### Agent status API

GET https://hook.simplehook.dev/api/agent/status
Authorization: Bearer ak_...

Returns listener count, queue health, cursor positions, and per-route breakdown.
